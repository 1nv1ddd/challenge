# AIChatHub — FAQ для пользователей

База знаний AI-ассистента поддержки. Если ответа здесь нет — нажми «❌ Не помогло» под ответом бота, и тикет уйдёт оператору.

## Что это вообще

### Что такое AIChatHub?
Веб-чат и Telegram-бот с AI-ассистентом. Под капотом: RAG (поиск по документации), несколько моделей на выбор (облачные через RouterAI и локальные через Ollama), память диалога, MCP-инструменты, планировщик задач.

### Где открыть веб-версию?
Веб-чат на `http://138.124.86.111/`. Логин не требуется — это демо-инсталляция.

### Где Telegram-бот?
Найди в Telegram бота `@aiiisupport_bot` (имя может отличаться у вашей инсталляции) и нажми Start.

## Аккаунт и вход

### Нужно ли регистрироваться?
В демо-версии — нет. История чатов хранится локально в браузере (localStorage), переключение между чатами — через сайдбар.

### Ошибка «invalid token» при входе
В демо-версии токенов нет. Если такое сообщение пришло из API — значит истекла временная сессия RouterAI. Просто обнови страницу.

### Как сбросить всю историю?
В UI открой чат, нажми «Удалить чат» в сайдбаре. Или через DevTools: `localStorage.clear()` и обнови страницу.

## Использование чата

### Какие модели доступны?
- **GPT-4o Mini** — облачная (RouterAI), отвечает быстро (1-3 секунды).
- **Vikhr-Nemo-12B-Instruct (Q4_K_M)** — локальная, через Ollama. Доступна когда поднят SSH-туннель к локальной машине разработчика. Отвечает медленнее (10-60 секунд), но всё крутится у тебя.

### Что такое режим RAG?
Retrieval-Augmented Generation — перед тем, как сгенерировать ответ, модель ищет релевантные чанки в локальном индексе документации. Включается дропдауном «RAG»: `off`, `fixed`, `structural`, `compare`. В режиме `compare` — два ответа параллельно, для сравнения чанкингов.

### Что такое «температура»?
Параметр случайности модели. 0.1 — детерминированно (одинаковые ответы при повторе), 0.7 — стандартный чат, 1.2 — креативно/хаотично. Для технических вопросов держи 0.2-0.3.

### Что такое «Контекстная стратегия»?
Как обрабатывается длинная история диалога:
- **Sliding Window** — берётся последние N сообщений.
- **Branching** — можно создавать ветки разговора через checkpoints.
- **Facts** — модель сохраняет факты в working memory; история сжимается, факты остаются.

## Команды

### `/help <вопрос>`
Ассистент-разработчик, отвечает по нашему собственному коду и документации (README, `docs/`, исходники). Полезно когда ты сам пытаешься разобраться, как работает проект.

### `/support <вопрос>`
Ассистент поддержки (этот FAQ + handbook). Используется в Telegram-боте автоматически. В вебе тоже работает.

### `/admin <пароль>` (в Telegram-боте)
Открывает панель оператора со списком тикетов. Доступ только по паролю и только Telegram user_id'ам из переменной `ADMIN_TG_IDS`.

## API и интеграции

### Где документация API?
В файле `docs/api.md` репозитория. Краткая карта эндпоинтов: `/api/chat` (SSE-стрим), `/api/models`, `/api/memory/*`, `/api/rag/*`, `/api/scheduler/*`, `/api/mcp/*`.

### Лимиты на `/api/chat`
nginx режет до **10 запросов в минуту на IP** (`limit_req_zone 10r/m`, burst=2). Если получил `HTTP 429 Too Many Requests` — подожди минуту, квота восстановится.

### Длина контекста и ответа в Ollama
Настроены явно: `num_ctx=8192` (входной контекст), `num_predict=1024` (максимум токенов в ответе). Можно переопределить через env-переменные `OLLAMA_NUM_CTX` / `OLLAMA_NUM_PREDICT`.

## MCP

### Что такое MCP в этом проекте?
Model Context Protocol — способ дать LLM доступ к внешним функциям (tools). У нас работают:
- `git_mcp_server` — `get_current_branch`, `get_recent_commits`, `get_short_status` (используется в `/help`).
- `support_mcp_server` — `get_ticket`, `get_user`, `list_open_tickets` (используется в `/support` для подмеса данных тикета в промпт).

### Как подключить MCP-сервер в UI?
В шапке веб-чата — панель MCP, кнопка Connect, путь к скрипту (`/app/scripts/git_mcp_server.py` на VPS, локальный путь на dev-машине). После подключения LLM сама может звать tools во время чата.

## Поддержка

### Как связаться с оператором?
В Telegram-боте: задай вопрос, дождись AI-ответа, нажми «❌ Не помогло». Создастся тикет, оператор ответит здесь же. До получения ответа держи чат открытым.

### Как долго ждать ответа оператора?
Зависит от загрузки. В демо — на лучших усилиях.

### Где оператор смотрит мои тикеты?
В том же боте: пишет `/admin <пароль>`, видит список открытых тикетов, может ответить или закрыть. Каждый ответ оператора прилетает тебе в этот же Telegram-чат.
